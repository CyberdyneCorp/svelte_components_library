# Design Tokens

Design tokens are the foundational values that define the visual language of the Cyberdyne Design System. They ensure consistency across all components and products.

## Architecture

We use a **3-layer token architecture**:

```
Layer 1: Primitives    â†’  Raw values (colors, sizes)
Layer 2: Semantic       â†’  Named by purpose (backgrounds, text, borders)
Layer 3: Component      â†’  Scoped to components (button bg, input border)
```

### Why 3 layers?

- **Primitives** are stable â€” they rarely change
- **Semantic** tokens map intent â€” swap a theme by remapping these
- **Component** tokens enable surgical overrides without side effects

### Semantic Tokens

```css
/* Backgrounds */
--color-bg-primary      /* Main app background */
--color-bg-secondary    /* Cards, panels */
--color-bg-tertiary     /* Nested surfaces */
--color-bg-elevated     /* Elevated elements */

/* Text */
--color-text-primary    /* Main text */
--color-text-secondary  /* Supporting text */
--color-text-code       /* Code/terminal text (green) */
--color-text-link       /* Links (cyan) */

/* State */
--color-state-success   /* Green */
--color-state-warning   /* Amber */
--color-state-error     /* Red */
--color-state-info      /* Cyan */
```
