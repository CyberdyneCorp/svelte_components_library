# Getting Started

## Installation

The Cyberdyne component library is distributed as private packages via GitHub Packages.

### 1. Configure npm registry

Create or update your `.npmrc`:

```
@cyberdyne:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken=YOUR_GITHUB_PAT
```

### 2. Install packages

```bash
pnpm add @cyberdynecorp/svelte-ui-foundation @cyberdynecorp/svelte-ui-core
```

### 3. Import foundation styles

In your root layout or app entry point:

```svelte
<!-- +layout.svelte -->
<script>
  import "@cyberdynecorp/svelte-ui-foundation/styles";
</script>

{@render children()}
```

### 4. Use components

```svelte
<script>
  import { Button, Card, TextInput, Badge, DataTable, MarkdownEditor, MapView } from "@cyberdynecorp/svelte-ui-core";

  let query = $state("");
</script>

<Card variant="elevated">
  <TextInput
    bind:value={query}
    label="Search"
    placeholder="Search transactions..."
  />
  <Button variant="brand" onclick={() => search(query)}>
    Execute Query
  </Button>
  <Badge variant="success">Online</Badge>
</Card>

<!-- Rich markdown editing -->
<MarkdownEditor mode="split" enableMermaid />

<!-- Interactive map -->
<MapView markers={validatorNodes} />

<!-- Sortable data table -->
<DataTable rows={data} sortable paginated />
```

## Theming

All components use CSS custom properties. Override any token in your app:

```css
:root {
  /* Change the brand green to a different hue */
  --color-action-brand-default: #00e5ff;

  /* Adjust the background */
  --color-bg-primary: #050510;
}
```

## Typography

The design system uses three font families:

- **Space Grotesk** -- Display headings and titles
- **Inter** -- Body text and form inputs
- **JetBrains Mono** -- Code, labels, data values

Fonts are loaded via Google Fonts in the foundation CSS. For self-hosting, override the `--font-display`, `--font-body`, and `--font-mono` variables.

## Customization & Extensibility

Every component is designed as a **headless behavior shell** — zero business logic baked in. You inject behavior via callbacks, custom items, and CSS overrides.

### Callback Props

All interactive components expose `on*` callback props so you can inject your own logic without forking the component.

```svelte
<script>
  import { BlockEditor } from "@cyberdynecorp/svelte-ui-core";

  let markdown = $state("");

  function handleBlockCommit(block) {
    // Custom validation, analytics, auto-save — your logic here
    console.log(`Block ${block.id} committed:`, block.type);
    saveToServer(markdown);
  }

  function handleBlockDelete(block) {
    if (block.type === "heading") {
      // Prevent heading deletion with custom confirmation
      if (!confirm("Delete this heading?")) return;
    }
  }
</script>

<BlockEditor
  bind:value={markdown}
  onchange={(md) => debouncedSave(md)}
  onblockcommit={handleBlockCommit}
  onblockcreate={(block) => trackAnalytics("block_created", block.type)}
  onblockdelete={handleBlockDelete}
  onblockreorder={(blocks) => logReorder(blocks)}
/>
```

### Custom Toolbar Actions

Inject extra buttons into the toolbar with `toolbarActions`. Each action receives the active textarea element.

```svelte
<script>
  import { BlockEditor, insertMarkdown } from "@cyberdynecorp/svelte-ui-core";

  const extraTools = [
    {
      label: "Insert timestamp",
      icon: '<svg viewBox="0 0 24 24" width="16" height="16" fill="none" stroke="currentColor" stroke-width="2"><circle cx="12" cy="12" r="10"/><path d="M12 6v6l4 2"/></svg>',
      action: (textarea) => {
        const now = new Date().toISOString().split("T")[0];
        insertMarkdown(textarea, "", "", now);
      },
    },
    {
      label: "Wrap in callout",
      icon: '<svg viewBox="0 0 24 24" width="16" height="16" fill="none" stroke="currentColor" stroke-width="2"><path d="M12 2L2 7l10 5 10-5-10-5z"/><path d="M2 17l10 5 10-5"/></svg>',
      action: (textarea) => {
        insertMarkdown(textarea, "> **Note:** ", "", "your note here");
      },
    },
  ];
</script>

<BlockEditor toolbarActions={extraTools} />
```

## Accessibility

Every component in the Cyberdyne library is built with **WCAG 2.1 AA** compliance as a baseline. Accessibility is not a feature — it's a structural requirement. Here's what that means in practice, and how to use these components without breaking accessibility.

### What is Accessibility (a11y)?

Accessibility ensures that your application can be used by **everyone**, including people who:

- **Navigate with keyboards** — cannot use a mouse (motor impairments, power users)
- **Use screen readers** — rely on software that reads the interface aloud (blind or low-vision users)
- **Have color vision differences** — cannot distinguish certain color combinations
- **Are sensitive to motion** — experience discomfort from animations
- **Use assistive technologies** — switch controls, voice recognition, magnifiers

Failing to build accessible interfaces excludes ~15% of the world's population and may violate legal requirements (ADA, Section 508, EN 301 549).

### ARIA Patterns Used

Here's how the library applies ARIA across component categories:

**Dropdowns, Menus, Select**
```svelte
<!-- Trigger declares the popup relationship -->
<button aria-haspopup="listbox" aria-expanded={open}>
  Select option
</button>

<!-- Menu uses listbox/option roles -->
<div role="listbox">
  <button role="option" aria-selected={i === focusedIndex}>
    Option text
  </button>
</div>
```

**Tabs**
```svelte
<div role="tablist">
  <button role="tab" aria-selected={active === tab.key}>
    {tab.label}
  </button>
</div>
<div role="tabpanel">
  <!-- Panel content -->
</div>
```

**Modals & Dialogs**
```svelte
<div role="dialog" aria-modal="true" aria-labelledby="dialog-title">
  <h2 id="dialog-title">Confirm Action</h2>
  <!-- Focus is trapped inside while open -->
  <!-- Escape key closes the dialog -->
</div>
```

**Charts & Visualizations**
```svelte
<svg role="img" aria-label="Line chart showing model training loss over 50 epochs">
  <!-- SVG content is decorative; the aria-label provides the meaning -->
</svg>
```

**Live Regions (Toasts, Alerts)**
```svelte
<div role="alert" aria-live="assertive">
  Transaction confirmed successfully
</div>
```

### Best Practices When Using Components

**Always provide labels for inputs:**
```svelte
<!-- Good: label prop creates accessible association -->
<TextInput label="Email address" bind:value={email} />

<!-- Good: aria-label for icon-only buttons -->
<IconButton icon="search" aria-label="Search transactions" />

<!-- Bad: no label, screen reader says "edit text" -->
<TextInput bind:value={email} />
```

**Add context to icon-only actions:**
```svelte
<!-- Good: title prop adds tooltip + accessible name -->
<Button variant="ghost" title="Delete item" onclick={handleDelete}>
  <Icon name="trash" />
</Button>

<!-- Good: visually-hidden label -->
<button onclick={handleDelete}>
  <Icon name="trash" />
  <span class="sr-only">Delete item</span>
</button>
```

**Provide alt text for dynamic images:**
```svelte
<!-- In MarkdownPreview, images get alt from markdown syntax -->
<!-- ![Description of image](url) -->

<!-- When using Avatar, provide a name -->
<Avatar src={user.photo} alt="{user.name}'s profile photo" />
```

**Don't suppress focus rings:**
```css
/* Never do this */
*:focus { outline: none; }

/* The library uses :focus-visible which only shows for keyboard users */
/* If you must customize, replace don't remove: */
.my-button:focus-visible {
  outline: 2px solid var(--color-border-focus);
  outline-offset: 2px;
}
```

**Test with keyboard-only navigation:**
1. Unplug your mouse
2. Tab through the interface
3. Verify every interactive element is reachable
4. Verify focus order makes logical sense
5. Verify modals trap focus and Escape closes them

### Screen Reader Testing

For production apps, test with at least one screen reader:

- **macOS**: VoiceOver (built-in, `Cmd + F5`)
- **Windows**: NVDA (free) or JAWS
- **Mobile**: TalkBack (Android) or VoiceOver (iOS)

### Storybook A11y Addon

The library includes `@storybook/addon-a11y` which runs automated checks on every story. Look for the **Accessibility** tab in the Storybook panel to see violations, passes, and incomplete checks for each component.

### Block-Based Document Editor

Build a Notion-like document editor with custom slash commands and auto-save.

```svelte
<script>
  import { BlockEditor } from "@cyberdynecorp/svelte-ui-core";

  let content = $state("# My Document\n\nStart writing here...");
  let saveStatus = $state("saved");

  const customSlash = [
    { label: "Heading 1", type: "heading", template: "# ", icon: "H1", description: "Large heading" },
    { label: "Heading 2", type: "heading", template: "## ", icon: "H2", description: "Medium heading" },
    { label: "Bullet List", type: "unordered-list", template: "- ", icon: "•", description: "Unordered list" },
    { label: "Task List", type: "task-list", template: "- [ ] ", icon: "☐", description: "Checklist" },
    { label: "Code Block", type: "code-block", template: "```\n\n```", icon: "<>", description: "Code" },
    { label: "Callout", type: "blockquote", template: "> **Note:** ", icon: "!", description: "Info callout" },
    { label: "API Block", type: "code-block", template: "```http\nGET /api/v1/resource\nAuthorization: Bearer token\n```", icon: "{}", description: "API request" },
  ];

  let saveTimer;
  function autoSave(md) {
    saveStatus = "saving...";
    clearTimeout(saveTimer);
    saveTimer = setTimeout(async () => {
      await fetch("/api/documents/1", { method: "PUT", body: md });
      saveStatus = "saved";
    }, 1000);
  }
</script>

<BlockEditor
  bind:value={content}
  height="600px"
  slashMenuItems={customSlash}
  onchange={autoSave}
  onblockcommit={(block) => console.log("Committed:", block.type)}
/>
<small>{saveStatus}</small>
```

### Chat Interface

Compose a conversational AI interface with `ChatPanel`, `ChatResponse`, `Chatbox`, and `BotAnswer`.

```svelte
<script>
  import {
    Card,
    ChatResponse, Chatbox, BotAnswer, PromptExample
  } from "@cyberdynecorp/svelte-ui-core";

  let messages = $state([]);
  let isLoading = $state(false);
</script>

<Card style="display: flex; flex-direction: column; height: 600px;">
  <!-- Message history -->
  <div style="flex: 1; overflow-y: auto; padding: 1rem;">
    {#if messages.length === 0}
      <div style="display: grid; grid-template-columns: 1fr 1fr; gap: 0.5rem;">
        <PromptExample
          text="Explain how staking rewards work"
          onclick={() => sendMessage("Explain how staking rewards work")}
        />
        <PromptExample
          text="Show me the top DeFi protocols"
          onclick={() => sendMessage("Show me the top DeFi protocols")}
        />
      </div>
    {/if}

    {#each messages as msg}
      {#if msg.role === "user"}
        <ChatResponse variant="user">{msg.content}</ChatResponse>
      {:else}
        <BotAnswer content={msg.content} />
      {/if}
    {/each}
  </div>

  <!-- Input area -->
  <Chatbox
    placeholder="Ask Cyberdyne AI..."
    loading={isLoading}
    onsubmit={(text) => sendMessage(text)}
  />
</Card>
```
